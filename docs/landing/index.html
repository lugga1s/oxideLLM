7 Core Architectural Proofs

How oxideLLM eliminates the typical gateway bottleneck by writing code that respects computer hardware boundaries.

01

Telemetry Off the Critical Path

Uses a bounded lock-free MPSC queue. Telemetry events are offloaded in microseconds; client connections never wait for DB writes or log flushes.

// try_send is non-blocking
self.sender.try_send(event)
02

Zero-Copy SSE Forwarding

Chunks from upstream SSE are treated as raw byte streams. No JSON deserialization or string recreation in the hot loop.

// raw bytes are polled directly
Poll::Ready(Some(Ok(bytes)))
03

Micro-Batched Persistence

Events are grouped in memory and flushed asynchronously to disk (JSONL/Parquet) every 1000 items or 500ms, reducing disk IOPS by 99.9%.

if buffer.len() >= batch_size {
  flush_buffer(&mut buffer).await;
}
04

Active Health Monitoring

A lightweight background task pings upstreams dynamically, keeping a thread-safe healthy upstream pool in memory for routing.

// Check intervals in background
interval.tick().await
05

Client Disconnect Propagation

When the client closes the HTTP socket, the proxy cancels the upstream connection immediately, stopping useless token billing.

// Drop triggers instant cancel
impl Drop for StreamGuard
06

Aggressive Keep-Alive Pools

Bypasses TCP handshake latency using aggressive pre-warmed keep-alive pools and disabling Nagle's algorithm.

.pool_max_idle_per_host(1000)
.tcp_nodelay(true)
07

Zero Garbage Collection

No CPython, V8, or LuaJIT runtime. Memory is deterministic and reclaimed instantly without latency jitter or GIL lock contention.

// Pure RAII rust semantics
